# PRD: 보이스 그룹 카운슬링 앱

> 버전: v3.0 | 날짜: 2026-03-07 | 상태: 초안

---

## 1. 문제 정의

### 현재 상태 (AS-IS)
- 심리적 고민을 가진 성인들이 상담을 원하지만, 전문 상담은 비싸고 1:1 구조
- 시중 AI 챗봇은 결국 사람이 아님 — 공감과 연대감을 줄 수 없음
- BetterHelp 등 온라인 플랫폼도 매칭 알고리즘이 불투명하고 그룹 상담 없음 ([Forbes BetterHelp Review](https://www.forbes.com/health/mind/betterhelp-review/))

### 원하는 상태 (TO-BE)
- AI가 대화로 고민을 파악하고 → 비슷한 고민을 가진 사람들을 소그룹 매칭
- 자격증 있는 상담사(퍼실리테이터)가 그 그룹의 집단 상담을 진행
- 진짜 치유는 AI가 아닌 사람과 사람의 연결에서 온다

### 핵심 포지셔닝
> "AI 챗봇은 결국 AI다. 우리는 AI를 도구로 써서 사람과 사람을 연결한다."

### 성공 기준 (MVP)
- [ ] AI 챗봇이 대화 후 고민 주제를 정확히 분류한다
- [ ] 1주일 이내 그룹 + 상담사가 동시 매칭된다
- [ ] 사용자가 추천 그룹을 보고 납득 가능한 이유를 확인한다

---

## 2. 대상 사용자

> **고위험군(치료 필요)과 완전 건강한 사람 사이의 중간 지대**

- 일상은 기능하지만 내면의 회복이 필요한 성인
- 혼자 감당하기 버겁지만 정신과·심리상담까지는 아닌 것 같은 사람
- 다른 사람의 이야기를 들으면서 "나만 그런 게 아니구나" — 상호 회복이 가능한 사람
- 전문 용어: **Subclinical 수준** — 임상 진단 기준([DSM-5](https://www.psychiatry.org/psychiatrists/practice/dsm))에는 미충족하지만 심리적 지원이 필요한 상태

**서비스 경계 명시:** 본 서비스는 의료/임상 서비스가 아닌 정서 지원 플랫폼입니다.

## 3. 핵심 기능 (MVP)

### 3-1. AI 진단 챗봇 (방식 B)

> PHQ-9 / GAD-7 문항을 대화 속에 자연스럽게 녹여서 질문한다. 사용자에게 점수를 공개하지 않는다.

- 자유 대화로 시작 — 사용자가 먼저 말하게 함
- 대화가 막히면 AI가 열린 질문으로 유도 (혼합 방식)
- PHQ-9 전체(9문항) + GAD-7 전체(7문항) = **16문항을 대화에 녹임** (예상 15~20분)
- 스코어는 **내부 참고용** — 사용자에게 비공개, 그룹 배치 알고리즘에만 사용
- Claude API 기반 (claude-sonnet-4-6)

#### 대화 흐름

```
[자유 대화 — 호소문제 탐색]
    ↓
[PHQ-9 + GAD-7 16문항 대화에 녹임]
    ↓
[제외 기준 소프트 스크리닝]
    ↓
[그룹 준비도 탐색]
    ↓
[AI 분석 결과 생성]
```

#### 데이터 저장 (Supabase)
- ✅ 저장: AI 요약문 + PHQ/GAD 스코어 + 주제 분류 + 제외 플래그 + 준비도 점수
- ❌ 비저장: 대화 원문 (개인정보 리스크 최소화)

### 3-2. 위험도 스크리닝 & 제외 기준

#### 즉시 서비스 중단 기준

| 감지 항목 | 방법 | 대응 |
|----------|------|------|
| 자해/자살 위험 | PHQ-9 9번 문항 | 위기상담 전화(1393) 안내 + 상담자 즉시 알림 + 서비스 일시 중단 |

#### 집단 부적합 소프트 스크리닝 (상담자에게 플래그 전달)

| 제외 기준 | 이유 | 스크리닝 방법 |
|----------|------|-------------|
| 현재 물질 의존 | 그룹 역동 파괴, 다른 멤버에게 부정 영향 | 대화 중 음주/약물 패턴 탐색 |
| 반사회적 대인 패턴 | 다른 멤버를 조종·공격할 위험 | 대화 중 대인관계 패턴 탐색 |
| 급성 정신병적 증상 | 현실 검증력 저하 시 집단 참여 불가 | 대화 비일관성 감지 |

> 소프트 스크리닝은 AI가 감지 후 **상담자에게 플래그로 전달** → 상담자가 최종 판단

### 3-3. 그룹 배치 전략

#### 호소문제 카테고리 (5개 고정)

> 증상(symptom)이 아닌 **호소문제(presenting problem)** 기준으로 분류한다. Yalom(2020)의 집단 치유 요인 중 **보편성(Universality)** — "나만 그런 게 아니구나"는 구체적 상황이 비슷할 때 가장 강하게 작동한다.

| # | 카테고리 | 예시 상황 |
|---|---------|----------|
| 1 | 일·성과 압박 | "회사에서 버티기 힘들어요" |
| 2 | 대인관계 갈등 | "사람들과 자꾸 부딪혀요" |
| 3 | 자기 가치감 | "나는 뭘 해도 부족한 것 같아요" |
| 4 | 미래·진로 불확실성 | "앞으로 어떻게 해야 할지 모르겠어요" |
| 5 | 고립·외로움 | "주변에 마음 놓고 얘기할 사람이 없어요" |

#### 심각도 밴드 (인접 밴드 혼합 매칭)

> 동일 심각도 그룹은 위험할 수 있다. 인접 밴드를 혼합하면 **희망의 주입(Instillation of Hope)** — "저 사람은 나아지고 있구나" 효과가 생긴다.

| 밴드 | PHQ-9 스코어 | 성격 | 매칭 규칙 |
|------|-------------|------|----------|
| A | 0~9 (정상~경도) | 정서 지원 그룹 | A+B 혼합 가능 |
| B | 10~14 (중등도) | **핵심 타깃 (Subclinical)** | A+B 또는 B+C 혼합 가능 |
| C | 15~19 (중등도~중증) | 주의 관찰 그룹 | B+C 혼합 가능 |
| D | 20+ (중증) | **서비스 범위 외** | 전문 상담 연계 |

- **A+C 조합 금지** — 심각도 격차가 너무 크면 공감 형성 어려움
- GAD-7도 동일 밴드 구조 적용 (10점 기준 중등도)

#### 그룹 준비도 탐색

챗봇 대화 말미에 그룹 참여 동기를 자연스럽게 탐색한다:

> AI: "지금까지 이야기 나눠주셔서 감사해요. 혹시 비슷한 고민을 가진 분들과 이야기를 나눠본다면 어떨 것 같으세요?"

- 응답을 준비도 점수로 내부 변환
- 준비도 낮은 경우 → 상담자에게 플래그 전달

#### 그룹 크기

- **4~5명** (음성 전용 최적화 — 6명 이상은 발언 기회 감소, 침묵자 발생)
- 고정 멤버 구성 (매 세션 같은 멤버)

### 3-4. 상담자 승인

- AI 요약 1~2줄 + 추천 근거만 표시
- 제외 기준 플래그 해당 시 함께 표시
- 상담자가 **승인/반려** 최종 결정
- MVP(POC 단계): 상담자는 Supabase DB 직접 확인으로 대체

### 3-5. 집단 상담 세션
- 자격증 있는 상담사가 퍼실리테이터로 진행 (치료 개입 최소화)
- MVP: 상담사 직접 섭외 (지인 네트워크)
- 서비스 런칭 후: 상담사 플랫폼 등록 구조로 전환
- **세션 형태: 미정 (다음 결정 필요)**

## 4. POC 전략 (Wizard of Oz)

### 개념 설명

| 개념 | 정의 | 이 프로젝트에서 |
|------|------|----------------|
| **POC** (Proof of Concept) | 아이디어가 실제로 작동하는지 최소 비용으로 검증 | 챗봇 매칭 + 집단 상담이 가치 있는가 확인 |
| **Wizard of Oz** | 사용자는 자동화된 것처럼 느끼지만, 뒤에서 사람이 수동으로 운영 | 앱 없이 Discord로 운영하면서 실제 서비스처럼 경험시킴 |

> 핵심 차이: POC는 "되는가?" 검증, Wizard of Oz는 앱 개발 전 사람이 뒤에서 운영하는 실행 방식.

### POC 실행 흐름

```
[사용자 접속]
    ↓
[AI 챗봇 대화 (Claude API — 실제 구현)]
  ├─ 자유 대화 → 호소문제 탐색
  ├─ PHQ-9 + GAD-7 16문항 대화에 녹임
  ├─ 제외 기준 소프트 스크리닝
  └─ 그룹 준비도 탐색
    ↓
[AI 분석 결과 생성]
  ├─ 호소문제 분류 (5개 카테고리)
  ├─ 심각도 밴드 (A/B/C/D)
  ├─ 제외 플래그 (해당 시)
  └─ 준비도 점수
    ↓
[D밴드 or 자해/자살 위험 → 전문 상담 연계 + 서비스 중단]
    ↓
[그룹 추천 + 근거 요약 → 상담자에게 전달]
    ↓
[상담자: 제외 플래그 검토 + 최종 승인]
    ↓
[그룹 배치 확정 (4~5명)]
    ↓ (수동 — Wizard of Oz)
[Discord 비공개 채널 생성 + 초대]
    ↓
[자격증 상담사가 Discord 음성 채널에서 집단 상담 진행]
```

### MVP 구현 순서

```
Phase 1: 챗봇 (대화 → 분석 → DB 저장)
Phase 2: 상담자 승인 화면 (POC에서는 DB 직접 확인으로 대체)
Phase 3: 그룹 세션 (Discord Wizard of Oz)
```

### POC 목표

- [ ] 매칭 품질 검증 — 비슷한 호소문제끼리 잘 묶이는가?
- [ ] 심각도 밴드 검증 — 인접 밴드 혼합이 그룹 역동에 긍정적인가?
- [ ] 상담사 퍼실리테이터 모델 검증 — 집단 상담이 효과적인가?
- [ ] 사용자 반응 측정 — 앱으로 만들 가치가 있는가?

### POC 운영 계획

- 참여자 모집: 지인 네트워크
- 상담사: 지인 섭외 (자격증 필수)
- 규모: 미정

| 서비스 | 약점 | 우리의 차별화 |

| 오프라인 자조 모임 | 매칭 없음, 비구조적 | AI 매칭으로 유사 고민 그룹 구성 |

| 음성 (Phase 2) | Daily.co 또는 WebRTC | 카메라 없는 음성 전용 룸 |
